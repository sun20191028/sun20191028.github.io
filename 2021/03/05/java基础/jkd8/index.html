Lambda 表达式

  1. 允许将函数 作为一个方法的入参
  • lambda 表达式
  • 方法引用

    函数式接口 @FunctionalInterface

    函数式接口(Functional Interface)就是一个有且仅有一个抽象方法，但是可以有多个非抽象方法的接口。
    函数式接口可以被隐式转换为 lambda 表达式。

Predicate【filter】(一泛型入参 返回boolean）

  • DoublePredicate： 入参为double，返回boolean
  • IntPredicate： 入参为int，返回boolean
  • LongPredicate： 入参为long，返回boolean
  • BiPredicate: 两个泛型入参，返回boolean
  1. 一泛型入参 返回boolean
  2. 与操作
  3. 或操作
  4. 取反操作
  5. equal 判断

    Consumer【foreach】(一泛型入参 无返回)

  • BiConsumer : 两个泛型入参，无返回
  • DoubleConsumer： double入参，无返回
  • IntConsumer：Int入参，无返回
  • LongConsumer ： long入参，无返回
  • ObjDoubleConsumer：泛型入参+double入参，无返回
  • ObjIntConsumer：泛型入参+int入参，无返回
  • ObjLongConsumer：泛型入参+long入参，无返回
  1. 一泛型入参 无返回值
  2. 处理完前面那个处理后面的

Function【map】(一个泛型入参，一个泛型返回）

  • BiFunction : 两个泛型入参，一个泛型返回
  • DoubleFunction：一个dubbo入参，一个泛型返回
  • IntFunction： 一个int入参，一个泛型返回
  • LongFunction：一个long入参，一个泛型返回
  • DoubleToIntFunction 、DoubleToLongFunction
  • IntToDoubleFunction、IntToLongFunction
  • LongToDoubleFunction、LongToIntFunction
  • ToDoubleBiFunction、ToDoubleFunction
  • ToIntBiFunction、ToIntFunction
  • ToLongBiFunction、ToLongFunction
  1. 一个泛型入参，一个泛型返回。根据入参计算结果
  2. 先执行before,对返回的结果再进行当前计算。然后返回
  3. 先计算当前。再将得到的结果计算after

    Supplier(无入参，一泛型返回)

  • BooleanSupplier ：无入参，一boolean返回
  • DoubleSupplier：无入参，一double返回
  • IntSupplier: 无入参，一int返回
  • LongSupplier: 无入参，一long返回
  1. 无入参，泛型返回

默认方法

Java 8 新增了接口的默认方法
简单说，默认方法就是接口可以有实现方法，而且不需要实现类去实现其方法。
我们只需在方法名前面加个 default 关键字即可实现默认方法。
Java 8 的另一个特性是接口可以声明（并且可以提供实现）静态方法

  1. 引入default ，使得 Interface 的方法，也可以有方法梯
  2. default方法依然支持多态

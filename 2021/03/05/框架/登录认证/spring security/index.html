spring security 设计理念

spring security 基础知识点（spring实战第四版）

引入securty的过滤器，SecurityFilterChain （DefaultSecurityFilterChain）

  • spring security 是借助一系列 Filter 来提供各种安全功能，但这并不意味着我们需要在web.xml中配置大量的Filter、或者在WebApplicationInitializer（AbstractAnnotationConfigDispatcherServletInitializer）中添加大量的filter (WebApplicationInitializer 中添加filter 见 spring mvc 中的 Spitter 工程）

  • web.xml的 实现方式

  • WebApplicationInitializer 的实现方式

其实这两种方式，都是为了拉起 springSecurityFilterChain

  1. xml的DelegatingFilterProxy 是如何将 SpringSecurityFilterChain 中的Filter 注册进来
    xml 在 初始化的时候获取到 springSecurityFilterChain，然后在 doFilter中 走的 走的SecurityFilterChain 中的Filter
  2. AbstractSecurityWebApplicationInitializer的实现是，spring注解的方式启动会自动 运行 AbstractSecurityWebApplicationInitializer中的相应方法。然后将springSecurityFilterChain 封装到DelegatingFilterProxy 中，然后注册进来。

拦截请求定制

通过 http.authorizeRequests() 配置请求级别的安全性细节

.antMatchers(） 匹配的路径采用的ant风格的通配符
.regexMatchers() 匹配的正则表达式匹配的路径
access() SqEL 表达式计算结果为true，允许访问
authenticated() 允许 认证过的用户访问
permitAll() 无条件允许访问
hasAnyAuthority(String…) 用户具备 任意一个权限，允许访问
hasAnyRole(String… ） 用户具备 任意一个角色，允许访问
hasAuthority()
hasRole()
hasIpAddress() 允许给定的ip 访问
not 取反

@Override
protected void configure(HttpSecurity http) throws Exception {
    http
            .csrf()
                .disable() // 禁用 CSRF防护功能
            .authorizeRequests()
                .antMatchers("/toRegister","/toLoginPage", "/resource/**").permitAll()// 部分请求无需 授权
                .anyRequest().authenticated()
            .and()
            .formLogin()
                .loginPage("/toLoginPage")
                .loginProcessingUrl("/login")
                .permitAll()
            .and()
            .httpBasic();
}

如何将springmvc 嵌入tomcat的启动

思路

纯注解的spring mvc 是由SpringServletContainerInitializer 类拉起来的

那SpringServletContainerInitializer 是如何拉起来的?

StandardContext 中由SpringServletContainerInitializer调用 onStartup

initializers 中的 SpringServletContainerInitializer 是哪里来的？

ContextConfig 中进行添加的。那initializerClassMap 中又是哪里来的？

这里将 ServletContainerInitializer 的所有实现类拉起来了

并且 获取了 ServletContainerInitializer实现类的 @HandlesTypes注解

根据堆栈，往上查找，读过tomcat源码之后，就能知道HostConfig、StandardHost是干嘛的

tomcat 容器模型如下

host可以认为是一台主机提供的端口服务，里面发布了各种context
如/examples、/spring-test 等。
下图，就是开始将/spring-test 服务发布到tomcat容器（host）中

是如何找出上面的流程的

  1. 首先找到这一站，SpringServletContainerInitializer

  2. 然后通过方法调用栈找到是哪里调用了onStartup()，这里可以发现initializers 中包含了 SpringServletContainerInitializer的对象

  3. initializers是private，所以只能在此添加ContainerInitializer

  4. 反向查找调用，那肯定是上面ContextConfig调用的，不可能跑到spring中去

  5. 然后发现其来自于 initializerClassMap

  6. 最后 发现initializerClassMap在此处装载的数据

  7. 最后 ContainerInitializers 的结构如下
    以ContainerInitializer为key，其@HandlesTypes注解的类的实现类为value。通过 ContainerInitializer 一个个执行@HandlesTypes注解的类的实现类注意这里有一个AnnotationWebInitializer，这个就是我们自己的实现类，我们可以通过在这个类里重写方法，然后spring mvc注解启动的时候，就能将其拉起来

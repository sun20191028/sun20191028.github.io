JVM 内存布局

John Doe 2021-03-05 15:15:19
Categories： Tags：

运行时数据区

程序计数器

本地方法栈

  1. 单个线程指定的栈深（-Xss128k）太小时 StackOverflowError。
  2. 栈通常支持动态扩容，如果扩容时无法申请到更多的内存，OutOfMemoryError

虚拟机栈

方法区（非堆）

​ 永久代是永久代，方法区是方法区。永久代在堆里面，是jdk1.7 中堆的一种类型的内存名称， 只不过为了方便方法区的垃圾回收，因此采用永久代的垃圾回收方式，对方法区进行垃圾回收

jdk1.7之前用永久代(PermGen space)的方式实现，1.8之后才有元数据的方式实现，元数据(metaspace)不再保存在虚拟机中，而是保存在直接内存中

异常：java.lang.OutOfMemoryError: PermGen space

java堆

常量池

Class文件常量池：

包括 字面量 和 符号引用

运行时常量池：jdk1.8之前在方法区、1.8及 之后在元空间

字面量：简单点就是在java中，不需要通过new关键字来创建的一类数据。属于java的基本值，入short、int、long、byte、char、boolean、float、double，还有String ，创建这些变量的时候，不需要通过new

常量：任何具有不可变的东西我们都称之为常量

字符串是常量、不可修改问题。
String 被final修饰，只是限定了String 不能被继承，
关键属性：字符串数组 被private final修饰 只是说 该属性不能被外部引用，引起逃逸，在内部不能指向其他内存，但是其他对象获得该块内存的引用之后，依然可以对该块内存进行修改，从而达到修改该字符串的目的。而String在设计的时候，提供给外部调用的方法中，就限定了这样一种操作。

java中很多东西都是设计成不变的，比如：一个类的名字，一个类字段的名字/所属类型，一个类方法的名字/返回类型/参数名与所属类型，一个常量，还有在程序中出现的大量的字面值。

Java 中静态常量池 和 运行时常量池并非特指保存 final 常量，它还保存诸如字面量、类和接口全限定名、字段、方法名称、修饰符等永恒不变的东西。

字符串池：jdk1.7之前放在方法区，1.7 及之后放在堆

字符串池的实现-StringTable
在jdk1.7之前，String.intern() 会将字符串保存到方法区的字符串常量池中，容易引起内存溢出，所以将其移动到堆中

符号引用与直接引用

  1. 符号引用：
    在编译的时候，编译器分析java文件，将java中的 字面量、类和接口全限定名、字段、方法名称、修饰符 提取出来，形成一张常量表【相当于一个数组，通过下标引用】，此时对常量的编号就是符号引用。 在2个类中存在相同的字面量时，他们的符号引用可能不一样，不同类内部的符号引用，彼此没有关系
  2. 直接引用：直接引用可以是直接指向目标的指针

jvm-垃圾回收

John Doe 2021-03-05 15:15:19
Categories： Tags：

内存的分配

  1. 指针碰撞

  2. 空闲列表

如何判断为垃圾

  1. 引用计数法

    每次引用 计数加一，删除引用计数减一，但是对于句柄引用，引用对象间不断引用，无法回收

  2. 可达性分析法

    定义GCRoot ,一直由GCRoot 往下找，找得到的为有用对象，其他都为无效对象

如何回收

回收策略

  1. 标记清除算法

    内存空间零散，需要大空间时，空间不足，频繁促发gc

  2. 复制算法

  3. 标记整理算法

  4. 分代收集算法

垃圾回收器

  • XX:+UseSerialGC = Serial New (DefNew) + Serial Old
    小型程序。默认情况下不会是这种选项，HotSpot会根据计算及配置和JDK版本自动选择收集器
  • -XX:+UseParNewGC = ParNew + SerialOld这个组合已经很少用（在某些版本中已经废弃）
  • -XX:+UseConc(urrent)MarkSweepGC = ParNew + CMS + Serial Old
  • -XX:+UseParallelGC = Parallel Scavenge + Parallel Old (1.8默认) 【PS + SerialOld】
  • -XX:+UseParallelOldGC = Parallel Scavenge + Parallel Old
  • -XX:+UseG1GC = G1

Serial

  1. 单线程收集器
  2. 复制算法
  3. 新生代收集器
  4. 内存分配采用指针碰撞
  5. 可以配合 Serial Old 和CMS 老年代收集器使用

ParNew

  1. 多线程收集器，可以利用多核cpu
  2. 复制算法
  3. 新生代收集器
  4. 内存分配采用指针碰撞
  5. 可以配合 Serial Old 和CMS 老年代收集器使用

Parallel Scavenge

  1. 多线程收集器–可控的回收时间和吞吐量
  2. 复制算法
  3. 新生代收集器
  4. 内存分配采用指针碰撞
  5. 可以配合 Serial Old 和 Parallel Old 老年代收集器使用

Serial Old

  1. 单线程收集器
  2. 标记-整理算法
  3. 老年代收集器
  4. 新生代收集器 可以是 Serial 和 ParNew

Parallel Old

  1. 多线程
  2. 标记-整理算法
  3. 老年代收集器
  4. 新生代收集器 只能是Parallel Scavenge

CMS

  1. 初始标记-并发标记-重新标记-并发清除
  2. 标记-清除算法
  3. 老年代收集器
  4. 新生代收集器 可以是Serial 和 ParNew

当concurrent model fail 时 采用 Serial Old 收集器+ParNew

垃圾收集器组合

jdk1.7 默认垃圾收集器Parallel Scavenge（新生代）+Parallel Old（老年代）
jdk1.8 默认垃圾收集器Parallel Scavenge（新生代）+Parallel Old（老年代）
jdk1.9 默认垃圾收集器G1

Serial + Serial Old 适合 client 客户端使用（单核cpu）
Parallel Scavenge + Parallel Old 可控制的 回收时间 和 吞吐量
ParNew + CMS 可以利用 CMS的 低延迟

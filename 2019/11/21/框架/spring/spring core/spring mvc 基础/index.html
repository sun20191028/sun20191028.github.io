spring mvc 注解启动类

  1. 通常我们会把DispatcherServlet配置在web.xml中。然后web.xml会放置在 war包中
    tomcat 启动的时候一条路径是 加载web.xml

  2. servlet3.1 增强了之后，容器会在类路径中查找实现了javax.servlet.ServletContainerInitializer接口的实现类，如果发现了，会用它来配置servlet容器

  3. Spring提供了这个接口的实现类SpringServletContainerInitializer

    这个类通过注解，会查找WebApplicationInitializer 的实现类，

  4. spitter 就是引入了WebApplicationInitializer 的实现类
    AbstractAnnotationConfigDispatcherServletInitializer

    从类结构可以看出，其实spring只提供了Annotation(注解）的这种实现。

  5. 待补 加载包含Web组件的bean的 DispatcherServlet 应用上下文（控制器、视图解析器、处理映射器）

  6. 待补 要加载应用中的其他bean的 ContextLoaderListener 应用上下文
